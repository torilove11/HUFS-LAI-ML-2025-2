## 2. 데이터 구성
데이터는 총 100개의 강의계획서에서 추출한 텍스트로 구성되며, csv의 텍스트는 다음 7가지 feature로 표현됩니다.

| Feature | 설명 |
|--------|------|
| CourseTitle | 강의 교과명 |
| Classification | 전공(선택)/교양/기초 |
| CourseDescription | 교과목개요 및 학습목표(팀플 여부 판단을 위함) |
| AssignmentScore | 학습평가방식(100) 중 과제 비율 |
| EtcScore | 학습평가방식(100) 중 발표 및 토론, 프로젝트, 수업참여도 비율 |
| TeachingMethods | 수업운영방식(과제, 팀플 여부 판단을 위함) |
| OtherNotes | 기타안내 및 유의사항(위 목적과 동일) |

`CourseTitle`과 `Classification`은 필터링 시스템의 마지막 필터링을 위해 추출한 feature이며,
`AssignmentScore`, `EtcScore`는 학습평가방식에서 공식적으로 언급된 feature로 수치형 데이터이며, 머신러닝 때에 바로 사용가능합니다.
`CourseDescription`, `TeachingMethods`, `OtherNotes`에선 긴 길이의 문장형 텍스트로 추출하였기 때문에, 추후 전처리 과정에서
팀플, 과제의 빈도를 파악할 수 있는 텍스트를 분류 기준으로 점수를 선정할 계획입니다.

## 3. EDA 요약 결과
### 3-1. Target 변수 분포 : 배점 구조 분석
프로젝트 핵심인, '과제 부담'과 '팀플 유무'를 파악하기 위해 `AssignmentScore`와 `EtcScore`의 분포를 분석했습니다.
* `AssignmentScore`: 대체로 **10% ~ 30%** 구간에 집중되어 있으며,
  0인 강의도 일부 존재하나, 대부분 과제가 포함되어 있습니다.
* `EtcScore`:0인 강의가 많지만 **(팀플이 없을 확률 높음)**, 점수가 존재하는 경우 10%~20% 구간에 분포하며
최대 50%까지 할당된 강의도 있어 팀플 비중의 편차가 있음을 보입니다.

### 3-2. Classification 경향 분석
이는 기초 과목에 대한 데이터는 2건으로 너무 적다고 판단하여, 전공과 교양 두 종류로 `Classification`에 따라 배점 분포의 차이를 분석했습니다.
전공선택은 '전공'으로 표현했습니다.
* 산점도로 표현한 결과, 교양은 과제물 점수가 10~20% 구간에 밀집되는 경향이 드러났고,  전공 선택은 과제와 기타 배점이 교양보다 다양했습니다.
* 이는 전공 선택이 단순 과제보다는 프로젝트나 실습 등 다양한 평가 방식을 채택하고 있음을 나타낼 수 있지만, 데이터 양의 차이가 있어,
  더 자세하게 측정하기 위해선 전공 선택 수업에 대한 데이터가 더 필요합니다

### 3-3. 결측값 분석
데이터 수집 품질을 점검하기 위해 결측값 히트맵으로 분석했습니다.
* `AssignmentScore`와 `EtcScore` 등 수치형 데이터는 결측 없이 깨끗하게 수집되었습니다.
* 반면 `Classification`, `CourseDescription`, `TeachingMethods`, `OtherNotes` 등 텍스트 데이터는 모두 길이가 다르고,
* `OtherNotes`는 다수의 결측값이 확인됐습니다.
* 이는 해당 섹션(`OtherNotes`)이 비어있어, 결측값으로 남아있는 것으로 확인했습니다.
* 이러한 값은 팀플 및 과제에 대한 정보를 얻기 어렵기에, `OtherNotes`항목이 있는 데이터를 더 수집할 필요가 있어보입니다.

## 5. 추후 계획(데이터 추가 수집 및 전처리)
교양 과목 데이터가 전공선택, 기초 과목에 비하여 데이터가 많아, 과목 별 분포를 균등하게 맞추기 위하여 추가 수집이 필요해보입니다.
Target 변수인 `Assignment`가 대부분 10%~30%이기 때문에, 과제 평가 비율이 아닌 과제의 상대적인 양을 비교할 수단이 필요하다고 생각합니다.
따라서 `OtherNotes`에서 진행할 텍스트 분류를 위해 `OtherNotes`가 결측값이 아닌 계획서 데이터를 더 수집할 예정입니다.
- 1.  `EtcScore`가 0이 아닌 데이터를 다수 확보하였지만, Assignment3 에서 받은 피드백처럼 0인 데이터도 더 필요하다고 생각합니다. 더 높은 정확도로 분류하는 모델을 위해서
       데이터 종류를 균등하게 맞출 필요가 있습니다.
- 2.  수집된 텍스트 데이터를 팀플임을 예측가능한 텍스트를 분류하는 과정을 필요로 합니다. 또한 분류 횟수로 점수를 매겨
    과제 및 팀플 부담 점수를 부여한 데이터를 제작할 예정입니다.
